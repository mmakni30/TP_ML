def regression_logistique(X,t,Nitermax=20,eps_conv=1e-3):
    '''Entrees :
    X = [ones(N_train,1) x_train];
    t = class_train 
    Nitermax = nombre maximale d'itérations (20 par défaut)
    eps_conv = critère de convergence sur norm(w-w_old)/norm(w) ; 
    eps_conv=1e-3 par défaut
    
    Sorties : 
    w : vecteur des coefficients de régression logistique
   Niter : nombre d'itérations utilisées effectivement
   
   Fonction de régression logistique pour la classification binaire.
   
   Utilisation :
       Nitermax = 50
       eps_conv = 1e-4
       [w,Niter] = regression_logistique(X,t,Nitermax,eps_conv)
    '''
    N_train = X.shape[0]

    #initialisation : 1 pas de l'algorithme IRLS
    w = np.zeros((X.shape[1],))
    w_old = w 
    y = 1/2*np.ones((N_train,))
    R = np.diag(y*(1-y))   # diag(y_n(1-y_n))
    z = X.dot(w_old)-la.inv(R).dot(y-t)
    w = la.inv(X.T.dot(R).dot(X)).dot(X.T).dot(R).dot(z)

    # boucle appliquant l'algortihme de Newton-Raphson
    Niter = 1
    while ( (la.norm(w-w_old)/la.norm(w)>eps_conv) | (Niter<Nitermax) ):
        Niter = Niter+1
        y = 1/(1+np.exp(-X.dot(w))) # Apply sigmoid function to XwT
        R = np.diag(y*(1-y)) # construction of the diagonal matrix y(1-y)
        w_old = w 
        z = X.dot(w_old)-la.inv(R).dot(y-t) # calculate z using old values of w 
        w = la.inv(X.T.dot(R).dot(X)).dot(X.T).dot(R).dot(z) # calcul new w
         
    return w, Niter

Here we will apply our model on a data that contains outliers. They should be easier to classify as they are fare from the decision boundary, but this would mean that in our diagonal matrix containing the y(1 -y) there will be zeros, so the matrix might not be inversible

def regression_logistique_not_inv_R(X,t,Nitermax=20,eps_conv=1e-3, out=False):
    '''Entrees :
    X = [ones(N_train,1) x_train];
    t = class_train 
    Nitermax = nombre maximale d'itérations (20 par défaut)
    eps_conv = critère de convergence sur norm(w-w_old)/norm(w) ; 
    eps_conv=1e-3 par défaut
    
    Sorties : 
    w : vecteur des coefficients de régression logistique
   Niter : nombre d'itérations utilisées effectivement
   
   Fonction de régression logistique pour la classification binaire.
   
   Utilisation :
       Nitermax = 50
       eps_conv = 1e-4
       [w,Niter] = regression_logistique(X,t,Nitermax,eps_conv)
    '''
    N_train = X.shape[0]

    #initialisation : 1 pas de l'algorithme IRLS
    w = np.zeros((X.shape[1],))
    w_old = w 
    y = 1/2*np.ones((N_train,))
    R = np.diag(y*(1-y))  # diag(y_n(1-y_n))
    z = X.dot(w_old)-la.inv(R).dot(y-t)
    
    w = la.inv(X.T.dot(R).dot(X)).dot(X.T).dot(R).dot(z)

    # boucle appliquant l'algortihme de Newton-Raphson
    Niter = 1
    while ( (la.norm(w-w_old)/la.norm(w)>eps_conv) | (Niter<Nitermax) ):
        Niter = Niter+1
        y = 1/(1+np.exp(-X.dot(w)))
        R = np.diag(y*(1-y))  
        w_old = w        
        try: # if the R is not inversible, we stop
            inv_R = la.inv(R)
        except:
            if out:
                print(f" R is not inversible after {Niter}")

            break
        z = X.dot(w_old)-inv_R .dot(y-t)
        w = la.inv(X.T.dot(R).dot(X)).dot(X.T).dot(R).dot(z)
         
    return w, Niter

Since the estimation of the coefficients is wrong, we didn't plot the decision boundary but chose to plot the misclassified observations. Now class 1 was well classified. From the coefficients array we can notice that there is a big gap between the weight correponding to feature 1 and the intercept and that of the second features (which is a negative value).

Class 2 contains mostly element whose first feature coordinate is negative, which means $-w_1x_{k,1}$ in the probability expression for class 1 will make it so that $P(x_k \in C1)$ converges to 0, but we should have the opposite effect with elements of class 1 (positive first coordinate) which is not the case and that might be due to the intercept that is overset.

Logistic Regression models are not much impacted due to the presence of outliers because the sigmoid function tapers the outliers. But the presence of extreme outliers may somehow affect the performance of the model and lowering the performance. In general, when we have extreme outliers and we want to use Logistic regression, we have to remove these points. The idea of removing the extreme misclassified data points are being implemented in the following steps:

  1. Train a logistic regression model for a given dataset
  2. Compute the weight vector for the model trained in step 1. In scikit-learn, the weight vector can be computed using classifier.coef_.
  3. Compute the distance of each of the data points from the weight vector.
  4. Decide a distance threshold, to remove the extreme data points.
  5. Remove the data points having a distance greater than the decided threshold.
  6. Re-train the same logistic regression model on the new data from step 5.

Adjusting the classification threshold¶

We have two times the number of obersavtions belonging to class 2 than to class 1, and this affect the calculations of the intercept in our Logistic model. One way to counter this problem, is to apply a weighted class intercept, as implemented in scikit-learn, and we will explore this part afterwards. Another way is to adjust the classification threshold, up to now we took the maximum of probabilities. That is to say we compared probabilities to 0.5. However, and especially in imbalanced cases, a different threshold might work better, by helping the classification of the minority class.

In this part, we will take a look at the probability distribution and plot the ROC curve to get the best threshold (maximizing the G-mean score, $g-mean = \sqrt{TPR*(1-FPR)}$)

Most probabilities are distributed around 0 given the class imbalance, so it makes sense that the best threshold we be lower than 0.5, and in this case 0.27. Usually theROC curve could be optimistic and doesn't take into account the performance on the minority class. We will use the F1-MCC curve that is more suitable for imbalanced data such as ours [4], the best threshold will be the closest to the point (1, 1):

from sklearn.metrics import f1_score, matthews_corrcoef, roc_curve
# function to compare the probabilities with a threshold given as an argument
decision_thres = lambda x, thres: np.array([thres<=sigmoid(np.dot(x, w.T))]).astype(int)

F1 = np.zeros(len(thresholds))
MCC = np.zeros(len(thresholds))
# in order not to store a matrix of prediction for each threshold, 
# we'll iterate over the thresholds
for idx, thres in enumerate(thresholds):
    y_pred = decision_thres(X, thres).reshape(t.shape)
    F1[idx] = f1_score(t, y_pred)
    MCC[idx] = matthews_corrcoef(t, y_pred)
best = np.array([1, 1])
best_thres = np.argmin([la.norm(best -x) for x in np.hstack((F1, MCC))])

plt.ylabel("MCC")

def plot_data(lda, X, y, y_pred, fig_index):

        plt.title("Linear Discriminant Analysis")
        plt.ylabel("Data with\n fixed covariance")

        plt.ylabel("Data with\n varying covariances")

    tp = y == y_pred  # True Positive

    X0, X1 = X[y == 0], X[y == 1]
    X0_tp, X0_fp = X0[tp0], X0[~tp0]
    X1_tp, X1_fp = X1[tp1], X1[~tp1]

def plot_ellipse(splot, mean, cov, color):
    v, w = linalg.eigh(cov)
    u = w[0] / linalg.norm(w[0])
    angle = np.arctan(u[1] / u[0])
    angle = 180 * angle / np.pi  # convert to degrees
    # filled Gaussian at 2 standard deviation
    ell = mpl.patches.Ellipse(
        mean,
        2 * v[0] ** 0.5,
        2 * v[1] ** 0.5,
        180 + angle,
        facecolor=color,
        edgecolor="black",
        linewidth=2,
    )
    ell.set_clip_box(splot.bbox)
    ell.set_alpha(0.2)
    splot.add_artist(ell)
    splot.set_xticks(())
    splot.set_yticks(())


def plot_lda_cov(lda, splot):
    plot_ellipse(splot, lda.means_[0], lda.covariance_, "red")
    plot_ellipse(splot, lda.means_[1], lda.covariance_, "blue")

def regression_logistique_regularized(X,t,Nitermax=20,eps_conv=1e-3, Lambda = 1e-5):
    '''Entrees :
    X = [ones(N_train,1) x_train];
    t = class_train 
    Nitermax = nombre maximale d'itérations (20 par défaut)
    eps_conv = critère de convergence sur norm(w-w_old)/norm(w) ; 
    eps_conv=1e-3 par défaut
    Lambda = 1e-5 facteur de régularisation
    
    Sorties : 
    w : vecteur des coefficients de régression logistique
   Niter : nombre d'itérations utilisées effectivement
   
   Fonction de régression logistique pour la classification binaire.
   
   Utilisation :
       Nitermax = 50
       eps_conv = 1e-4
       Lambda = 1e-5
       [w,Niter] = regression_logistique(X,t,Nitermax,eps_conv)
    '''
    N_train = X.shape[0]

    #initialisation : 1 pas de l'algorithme IRLS
    w = np.zeros((X.shape[1],))
    w_old = w 
    y = 1/2*np.ones((N_train,))
    R = np.diag(y*(1-y)) + Lambda*np.identity(len(y)) # diag(y_n(1-y_n))
    z = X.dot(w_old)-la.inv(R).dot(y-t)
    
    w = la.inv(X.T.dot(R).dot(X)).dot(X.T).dot(R).dot(z)

    # boucle appliquant l'algortihme de Newton-Raphson
    Niter = 1
    while ( (la.norm(w-w_old)/la.norm(w)>eps_conv) | (Niter<Nitermax) ):
        Niter = Niter+1
        y = 1/(1+np.exp(-X.dot(w)))
        R = np.diag(y*(1-y))  
        w_old = w 
        inv_R = la.inv(R+2*Lambda*np.eye(R.shape[0]))
        z = X.dot(w_old)-inv_R .dot(y-t)
        w = la.inv(X.T.dot(R).dot(X)).dot(X.T).dot(R).dot(z)
         
    return w, Niter

Commentaires :

The misclassification dropped from 25 and 44 to 15 and 18 respectively after tuning the model along the regularization parameter, the solver and the penalty. The error on the training set is still 0 which is odd, as usually the regularization decreases the performance on this set. Instead of calculating the Hessian matrix (like in our algorithm) and finding non inversible matrices if some features are colinear or some observations are far from the decision line, sci-kit learn uses another solver that approximate this matrix for the optimization part. The class activation map shows that the edges on the left play the biggest role in the classification, which makes since since that is the part the differentiate between a 5 and a 6. Most of the other pixels are not used as they don't add relevant information to this classification task.

In their documentation, liblinear works better for small datasets which is our case (around 300 samples) and the l1 penalty was better which disproves our claim earlier. However, it could be that this penalty is better for this optimization algorithm, and might not work as well for others.

This algorithm was able to classify well images in less than 0.1s and without calculation accelerators (like GPU). Usually in this task, the first model we use is a deep learning CNN model which needs a lot more samples to estimate its parameters, and it needs a lot more time to fit to the training data. Our approach should be to first look at the data available, and try simple and interpretable models like the logistic regression before going to more complicated ones.
